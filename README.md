# Приложение для тестирования студентов: Сборка изображений

Веб-приложение для проведения интерактивных тестов, где студенту необходимо собрать изображение из фрагментов (пазл) и выбрать правильное название объекта. Результат оценивается на основе правильности ответа и затраченного времени.

## Общее описание

Приложение представляет собой одностраничное веб-приложение (SPA), работающее полностью на стороне клиента (frontend). Оно загружает данные теста из внешнего XML-файла по ссылке, что позволяет легко менять задания без изменения кода программы.

Основная задача студента:
1. Ввести свои данные (Фамилия, Имя).
2. Собрать перемешанное изображение, меняя фрагменты местами.
3. Прочитать задание и выбрать правильный вариант ответа.
4. Получить оценку, зависящую от времени сборки и правильности выбора.

## Возможности

- **Динамическая загрузка теста:** Тест загружается через URL-параметр, что позволяет создавать ссылки на конкретные варианты тестирования.
- **Интерактивный пазл:** Реализован механизм "Drag-and-Drop" (перетащи и оставь). Фрагменты меняются местами при перетаскивании одного на другой.
- **Гибкие настройки:** Для каждого вопроса можно задать размер сетки (rows, cols), текст задания, временные лимиты для оценок.
- **Режим "без подсказок":** Студент не видит, правильно ли собран пазл, до момента нажатия кнопки "Подтвердить ответ".
- **Фиксация результатов:** Выводится детальная статистика: количество правильных ответов, средняя оценка, время прохождения, а также разбор каждого вопроса.

## Запуск приложения

Для работы приложения не требуется серверная часть (backend). Достаточно разместить файл `index.html` на любом веб-сервере или открыть локально (с ограничениями CORS).

### Способ 1: Локальный запуск
Просто откройте файл `index.html` в браузере.
*Примечание:* Если XML-файл находится на удаленном сервере, могут возникнуть ошибки CORS (Cross-Origin Resource Sharing). Рекомендуется использовать локальный сервер (например, Live Server в VS Code).

### Способ 2: Размещение на сервере
1. Загрузите файл `index.html` на ваш хостинг.
2. Загрузите XML-файл с тестом в доступное место (на тот же домен или на сторонний ресурс с разрешенным CORS).
3. Сформируйте ссылку для студентов.

**Формат ссылки:**
```
https://ваш-сайт.ru/index.html?file=https://путь-к-файлу/test.xml
```

## Формат XML-файла

Тест описывается в формате XML. Структура должна строго соответствовать схеме.

### Структура

```xml
<test>
    <question>
        <image>URL_изображения</image>
        <rows>3</rows>
        <cols>3</cols>
        <timeExcellent>60</timeExcellent>
        <timeGood>120</timeGood>
        <task>Текст задания, которое увидит студент над ответами.</task>
        <answers correct="0">
            <answer>Правильный ответ (индекс 0)</answer>
            <answer>Неправильный ответ 1</answer>
            <answer>Неправильный ответ 2</answer>
        </answers>
    </question>
    <!-- Можно добавить сколько угодно вопросов <question> -->
</test>
```

### Описание полей

| Поле | Обязательное | Описание |
| :--- | :---: | :--- |
| `<image>` | Да | Прямая ссылка на изображение (JPG, PNG). Должно быть доступно по HTTP/HTTPS. |
| `<rows>` | Нет | Количество строк в сетке пазла. По умолчанию `3`. |
| `<cols>` | Нет | Количество столбцов в сетке пазла. По умолчанию `3`. |
| `<timeExcellent>` | Нет | Время в секундах, за которое ставится оценка "Отлично" (при правильном ответе). По умолчанию `60`. |
| `<timeGood>` | Нет | Время в секундах, за которое ставится оценка "Хорошо". Должно быть больше `timeExcellent`. По умолчанию `120`. |
| `<task>` | Нет | Текст задания. Выводится в верхней части экрана тестирования. Если не указано, выводится текст по умолчанию. |
| `<answers>` | Да | Контейнер для вариантов ответов. Атрибут `correct` указывает на индекс правильного ответа (нумерация с 0). |
| `<answer>` | Да | Текст варианта ответа. |

## Система оценивания

Оценка за каждый вопрос формируется по следующим правилам:

1. **Проверка сборки:** Изображение должно быть собрано правильно (каждый фрагмент стоит на своем месте).
2. **Проверка ответа:** Должен быть выбран правильный вариант ответа.

### Итоговая оценка за вопрос:
- **5 (Отлично):** Пазл собран И ответ верен И время ≤ `timeExcellent`.
- **4 (Хорошо):** Пазл собран И ответ верен И `timeExcellent` < время ≤ `timeGood`.
- **3 (Удовлетворительно):** Пазл собран И ответ верен И время > `timeGood`.
- **2 (Неудовлетворительно):** Пазл собран неверно ИЛИ выбран неверный ответ.

### Итоговая оценка за тест:
Рассчитывается как среднее арифметическое оценок за все вопросы.
- Средний балл ≥ 4.5 → "Отлично"
- Средний балл ≥ 3.5 → "Хорошо"
- Средний балл ≥ 2.5 → "Удовлетворительно"
- Иначе → "Неудовлетворительно"

## Пользовательский интерфейс

### 1. Экран регистрации
- Поля ввода Фамилии и Имени.
- Информация о необходимости передать XML-файл через URL.
- Кнопка "Начать тестирование".

### 2. Экран тестирования
- **Заголовок:** Номер вопроса, круговой таймер с индикацией скорости (зеленый/желтый/красный), кнопка "Выход".
- **Прогресс-бар:** Показывает продвижение по тесту.
- **Блок задания:** Выводится текст из тега `<task>`.
- **Область пазла:** Сетка с перемешанными фрагментами. Кнопка "Перемешать" для случайной пересборки.
- **Варианты ответов:** Список кнопок для выбора правильного названия.
- **Кнопка подтверждения:** Активируется только после выбора варианта ответа.

### 3. Экран результатов
- Данные студента (ФИО).
- Карточки со статистикой: Количество правильных ответов, Средняя оценка, Общее время.
- **Детализация:** Список всех вопросов с миниатюрой, статусом (Верно/Ошибка), оценкой, временем и текстом ответа студента.
- Кнопка "Пройти заново".

## Технические детали

- **Технологии:** HTML5, CSS3, JavaScript (ES6+), Tailwind CSS (via CDN).
- **Парсинг:** Используется встроенный `DOMParser` для чтения XML.
- **Работа с изображениями:** Используется CSS-свойство `background-position` в процентном соотношении для корректного отображения фрагментов пазла любой размерности.
- **Анимация:** Плавные переходы реализованы на CSS (`transition`, `keyframes`).
- **Безопасность:** Приложение работает на стороне клиента. XML-файл должен быть общедоступным по ссылке.
